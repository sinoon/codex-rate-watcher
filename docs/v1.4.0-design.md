# v1.4.0 — Global Hotkeys + CLI Tool + Raycast Extension

## Release Overview

v1.4.0 transforms Codex Rate Watcher from a menu bar utility into a full developer toolkit with three new access points:

1. **Global Hotkey (command+shift+K)** — Toggle the popover from anywhere
2. **`codex-rate` CLI** — Terminal-first monitoring with JSON output
3. **Raycast Extension** — Native Raycast integration via CLI bridge

## Architecture Changes

### Multi-Target Package Structure

```
Package.swift
+-- CodexRateKit (library)     <- shared models, API client, auth, estimator
+-- CodexRateWatcherNative     <- GUI app (depends on CodexRateKit)
+-- codex-rate                 <- CLI tool (depends on CodexRateKit)
```

The monolithic single-target was refactored into a shared library (`CodexRateKit`) consumed by both the GUI and CLI targets. This enables code reuse without duplication.

### Key Migrations to CodexRateKit
- `Models.swift` — All shared data types (UsageSnapshot, UsageLimit, etc.)
- `AuthStore.swift` — Auth file reader with JWT parsing
- `UsageAPIClient.swift` — API client for chatgpt.com/backend-api/wham/usage
- `UsageEstimator.swift` — Linear regression burn rate estimation
- `AppPaths.swift` — Shared path constants + UsageSample struct

### Swift 6 Sendable Compliance
All cross-module types required `Sendable` conformance:
- Value types: Direct `Sendable` conformance
- Types containing JSONDecoder: `@unchecked Sendable`

## Component: Global Hotkey

### Implementation: NSEvent Monitor
Selected NSEvent.addGlobalMonitorForEvents over Carbon HIToolbox:
- No C callback Sendable issues
- Clean @MainActor isolation
- No deinit complications

### Configuration
- Default: command+shift+K (Command + Shift + K)
- Persisted to: ~/Library/Application Support/CodexRateWatcherNative/hotkey-config.json
- Toggle via right-click menu on status bar icon

## Component: CLI Tool (`codex-rate`)

### Subcommands
| Command | Description | Key Flags |
|---------|-------------|-----------|
| `status` | Current usage quotas | `--json` |
| `profiles` | List auth profiles | `--json` |
| `watch` | Continuous monitoring | `--interval <s>`, `--json` |
| `history` | Usage sparklines | `--hours <n>`, `--json` |
| `help` | Usage information | — |

### Design Principles
- Zero external dependencies (hand-written argument parser)
- Unicode box-drawing for rich terminal output
- ANSI color with NO_COLOR / non-TTY detection
- JSON mode for all commands (machine-readable output for Raycast/scripts)

## Component: Raycast Extension

### Architecture
```
raycast-extension/
+-- package.json
+-- tsconfig.json
+-- src/
    +-- utils.ts       <- CLI execution + type definitions
    +-- status.tsx      <- Quota status (Detail view)
    +-- profiles.tsx    <- Profile list (List view)
    +-- history.tsx     <- Usage history (Detail view)
```

### CLI Bridge Pattern
The Raycast extension uses CLI-as-API:
1. Calls `codex-rate <cmd> --json`
2. Parses typed JSON response
3. Renders in native Raycast components

This avoids duplicating API logic in TypeScript while leveraging Raycast rich UI.

### CLI Path Resolution
Searches in order: /usr/local, ~/.local, /opt/homebrew, build output directories.
